# 2026-06-22 工作日志

## 开机打不开网页问题彻底解决

### 根本原因
Vite 的缓存目录 `node_modules/.vite` 损坏，导致服务器进程在跑、端口在监听，但实际请求返回 **404**。

### 修复流程
1. 杀掉所有旧 node 进程
2. 删除损坏的 Vite 缓存目录 `node_modules/.vite`
3. 重新启动服务器 → 返回 200，页面正常

### 防止复发措施
- 已更新桌面 `启动网页服务器.bat`，每次启动时自动：
  1. 清理旧 node 进程
  2. 清除 Vite 缓存
  3. 启动服务器
- 以后遇到打不开的情况，直接双击桌面快捷方式重启即可。

### 备注
此方案解决了开机或长时间未使用后页面无法访问的问题，保证每次启动都是干净环境。

## Shader 组件报错永久解决

### 问题
开机后页面虽然能加载，但控制台报 `Shader components must be used inside an <Shader>` 错误，导致 React 渲染崩溃、页面空白。

### 根本原因
- 旧 Vite 缓存或 `dist/build` 产物中包含错误编译的 chunk
- `shaders/react` 版本可能不一致或未锁定，导致子组件校验失败

### 永久解决步骤
1. **代码层**：确认 `App.tsx` HeroSection 中 Shader 子组件已正确包裹在 `<Shader>` 内（已完成）
2. **缓存层**：升级 `startup-server.bat`，启动时清理 `node_modules/.vite`、`dist`、`build`
3. **版本层**：锁定 `shaders` 包版本为 `2.5.130`，删除 `node_modules` 与 `package-lock.json` 并执行 `npm install` 全新安装
4. **验证**：无痕模式访问 `http://localhost:3000`，确保无 Shader 报错且动效正常

### 效果
三重保障后，开机自启不再出现 Shader 组件报错，页面稳定渲染。

## 开机自启浏览器报 chrome-error 问题解决

### 现象
手动启动服务正常，但开机任务启动后，浏览器访问 http://localhost:3000 报：
> Unsafe attempt to load URL http://localhost:3000/ from frame with URL chrome-error://chromewebdata/

页面无法加载。

### 根本原因
- 服务本身启动成功（VITE ready in xxx ms）
- 但开机任务执行环境与手动运行不同：工作目录、权限、网络就绪时机、防病毒拦截等因素导致浏览器实际访问失败，Chrome 进入错误页面并触发安全策略拦截。

### 验证
手动执行 `node node_modules\vite\bin\vite.js --host 0.0.0.0 --port 3000` 可正常访问，确认代码与依赖无误。

### 解决方向
1. 任务计划中设置正确的“起始于”目录
2. 延长延迟启动时间（≥60 秒）
3. 使用最高权限运行
4. 检查防病毒软件是否拦截 node 进程
5. 确保网络与 localhost 解析在启动时已就绪

### 结论
此为环境差异导致的浏览器访问失败，非代码或依赖问题。修正任务计划环境即可根治。
